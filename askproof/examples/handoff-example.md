# Handoff Example

## User Input

```text
我要新开对话，帮我生成交接说明。
```

## AskProof Output

```text
当前判断：Handoff Memory｜交接记忆

用户真正的问题：
你担心新对话丢失上下文，希望下一个 AI 能接着做，而且不要重复踩坑。

当前任务目标：
让虚构项目 TinyInvoice 支持导出 PDF 发票。

已完成内容：
- 添加了“导出 PDF”按钮。
- 将按钮连接到 PDF 导出函数。
- AI 声称修复了 PDF 渲染依赖问题。

已验证内容：
- 暂无验证证据。

未验证内容：
- 点击按钮是否真的下载 PDF。
- PDF 中金额、税费和客户信息是否正确。
- 修改是否影响发票列表页。

当前风险：
- 当前只有“已修改”信息，没有“已验证”证据。
- 如果新对话直接继续扩展功能，可能会在未确认的基础上叠加问题。

关键决策：
- 先验收导出 PDF 主流程，再讨论批量导出或模板美化。

不要重复踩坑：
- 不要只接受“应该可以了”。
- 不要在没有截图或验证结果前继续加新功能。
- 不要大范围重构无关页面。

下一轮可复制 Prompt：
请根据这份交接继续。当前目标是让 TinyInvoice 支持导出 PDF 发票。请先复述已完成、已验证、未验证和风险。不要直接加新功能；先给出最小验证步骤，证明点击“导出 PDF”后能下载内容正确的 PDF。

你现在最应该问 AI 的一句话：
请根据这份交接继续，先验证导出 PDF 主流程，不要直接扩展新功能。
```
