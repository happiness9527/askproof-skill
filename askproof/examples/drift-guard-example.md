# Drift Guard Example

## User Input

```text
既然登录修好了，那顺便把会员中心、积分系统和优惠券也一起做了吧。
```

## AskProof Output

```text
当前判断：Drift Guard｜防跑偏

用户真正的问题：
你想继续推进功能，但登录修复是否真的完成还没有验收证据。

AI 当前说法是否可信：
当前不能判断登录是否可信，因为你没有提供“已验证”的证据。最多只能认为 AI 可能已经修改过登录相关内容。

已确认的信息：
- 你准备从登录问题扩展到会员中心、积分系统和优惠券。

未确认的信息：
- 登录是否真的修好。
- AI 是否验证过登录成功、登录失败、退出登录和刷新页面后的状态。
- 新功能会不会依赖尚未验证的登录状态。

风险点：
- 如果现在继续扩功能，后续问题可能来自登录、会员中心、积分或优惠券，很难定位。
- 新功能会建立在未验证的基础上，返工成本会变高。

下一步最稳妥动作：
先收口登录验收，再决定是否进入会员中心。

可直接复制给 AI 的 Prompt：
请先不要新增会员中心、积分系统或优惠券。请回到登录修复本身，按“已修改 / 已验证 / 未验证 / 风险”说明当前状态，并给出一个最小验收步骤，证明登录功能真的可用。登录通过验收后，再拆分下一阶段需求。

你现在最应该问 AI 的一句话：
请先停止扩展新功能，证明登录修复已经通过验收，再拆分会员中心、积分和优惠券的下一阶段计划。
```
