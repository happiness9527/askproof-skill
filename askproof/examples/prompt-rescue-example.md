# Prompt Rescue Example

## User Input

```text
还是不行，你自己解决。
```

## AskProof Output

```text
当前判断：Prompt Rescue｜低质量反馈救援

用户真正的问题：
你想让 AI 继续排查，但现在这句话没有提供足够信息，AI 可能会盲目改代码。

AI 当前说法是否可信：
当前没有新的 AI 说法可判断；重点是把你的反馈改成可执行的排查 Prompt。

已确认的信息：
- 当前结果仍然不符合预期。
- 你暂时没有提供具体失败现象。

未确认的信息：
- 哪里不行。
- 你看到的错误是什么。
- 上一轮修改是否有任何效果。
- 是否有截图、红色错误文字或 AI 最后一段回复。

风险点：
- 继续说“你自己解决”会让 AI 扩大修改范围，可能引入更多问题。

下一步最稳妥动作：
让 AI 先收窄问题，再决定是否需要你提供最低门槛证据。

可直接复制给 AI 的 Prompt：
请不要继续盲目改代码。请先按以下步骤排查：
1. 复述当前失败现象；
2. 说明你上一轮修改了哪些文件；
3. 判断问题可能发生在哪个环节；
4. 告诉我需要提供的最小证据；
5. 在没有新证据前，不要大范围重构；
6. 给出下一次修改后的验证方式。

你现在最应该问 AI 的一句话：
请不要继续盲目改代码，先复述失败现象、说明上一轮改了什么、告诉我你需要的最小证据，并给出下一次修改后的验证方式。
```
