# Evidence Ladder Example

## User Input

```text
我看不懂这个报错，也不知道怎么提供日志。
```

## AskProof Output

```text
当前判断：Evidence Ladder｜证据阶梯

用户真正的问题：
你不是不配合排查，而是不知道哪些信息对 AI 有用。

AI 当前说法是否可信：
当前没有足够信息判断。先收集最低门槛证据。

已确认的信息：
- 你看到了报错。
- 你不确定日志是什么，也不知道该复制哪一段。

未确认的信息：
- 报错出现在哪个页面或步骤。
- 报错最后一行写了什么。
- AI 上一轮是否说已经修复。

风险点：
- 如果直接让 AI “继续修”，它可能会猜问题，而不是根据证据定位。

下一步最稳妥动作：
你不用懂日志，先任选下面一种证据发回来：
1. 当前页面截图；
2. AI 最后一段“已完成/已修复”的回复；
3. 页面或终端里最后一行红色错误文字；
4. 你看到的异常现象，用普通话描述也可以。

可直接复制给 AI 的 Prompt：
我不懂日志。请告诉我你最需要哪一种最低门槛证据：截图、AI 最后一段回复、最后一行红色错误文字，还是我用普通话描述现象？在我提供前，请不要大范围修改。

你现在最应该问 AI 的一句话：
我不懂日志，请告诉我你最需要哪一种最低门槛证据，并在拿到证据前不要大范围修改。
```
