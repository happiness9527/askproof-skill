# Done Check Example

## User Input

```text
AI 说：“已修复，现在应该可以了。”
```

## AskProof Output

```text
当前判断：Done Check｜完成验收

用户真正的问题：
你不是想听 AI 再说一次“修好了”，你是想知道它有没有证据证明真的修好了。

AI 当前说法是否可信：
完成可信度：低。它只给了结论，没有给验证过程或结果。

当前状态：
AI 声称已修改，但未验证；不建议继续扩展功能。

已确认的信息：
- AI 声称问题已经修复。

未确认的信息：
- 它改了哪些文件。
- 它是否运行过项目。
- 原来的问题是否消失。
- 是否有截图、测试结果或手动验收记录。

风险点：
- 如果现在继续加功能，后面出错时很难判断问题来自旧问题还是新修改。

下一步最稳妥动作：
先让 AI 提供完成证据。

可直接复制给 AI 的 Prompt：
请先不要继续扩展功能。请按“已修改 / 已验证 / 未验证 / 风险”四部分说明：你改了什么、如何验证、验证结果是什么、还有哪些内容没有验证。如果没有验证，请直接标注“未验证”，不要说应该可以。

你现在最应该问 AI 的一句话：
请先证明这次修复已经生效，说明你改了什么、如何验证、验证结果是什么，以及哪些内容仍未验证。
```
