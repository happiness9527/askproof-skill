# Prompt Library

These prompts are designed for non-engineers. Copy one and send it to your AI agent.

## When AI Says “Fixed”

```text
请先不要继续扩展功能。请按“已修改 / 已验证 / 未验证 / 风险”四部分说明：你改了什么、如何验证、验证结果是什么、还有哪些内容没有验证。如果没有验证，请直接标注“未验证”，不要说应该可以。
```

## When You Want To Say “Still Broken”

```text
请不要继续盲目改代码。请先复述当前失败现象，说明你上一轮修改了哪些文件，判断问题可能发生在哪个环节，告诉我需要提供的最小证据，并给出下一次修改后的验证方式。
```

## When You Do Not Understand The Error

```text
我不懂日志。请用非工程师能理解的语言解释这个错误对项目意味着什么，并告诉我你最需要哪一种最低门槛证据：截图、最后一段 AI 回复、最后一行红色错误文字，还是我用普通话描述现象。
```

## When Scope Is Expanding Too Fast

```text
请先停止新增功能，回到原目标。请列出上一轮已修改内容、已验证内容、未验证内容和当前风险。只有当原目标通过验收后，再讨论新功能。
```

## When Opening A New Chat

```text
请根据当前对话生成交接说明，必须包含：当前任务目标、已完成内容、已验证内容、未验证内容、当前风险、关键决策、下一轮可复制 Prompt、不要重复踩坑的事项。
```

## Product Manager

```text
请先不要继续扩展功能。请按产品验收角度复述：目标用户是谁、原本期望的流程是什么、现在实际发生了什么、上一轮修改了哪些内容、哪些验收标准已经满足、哪些还未验证。然后给出一个最小修改方案和对应验收方式。
```

## Designer

```text
请不要整页重做。请先从视觉验收角度列出当前页面的问题：层级、间距、对齐、颜色、字体、移动端适配分别是否存在明显问题。每次只改最影响观感的 1-2 个点，并说明修改后我应该截图检查哪里。
```

## Operations

```text
请先不要大范围改自动化流程。请按运营执行链路复述：触发条件是什么、输入数据是什么、哪一步失败、成功时应该出现什么结果、上一轮修改了哪里、现在需要我提供的最小证据是什么。请先定位失败环节，再提出最小修复。
```
