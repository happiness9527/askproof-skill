# Use Cases

## 1. AI Says “Done”

Use AskProof to check whether the AI provided evidence or only a confident claim.

## 2. User Can Only Say “Still Broken”

Use AskProof to rewrite vague feedback into a useful prompt.

## 3. User Does Not Know Logs

Use AskProof to request the lowest-friction evidence first.

## 4. AI Gives A Technical Summary

Use AskProof to translate the summary into plain language and identify what remains unverified.

## 5. Scope Keeps Expanding

Use AskProof to stop drift and return to the original acceptance goal.

## 6. New Chat Or New Agent

Use AskProof to create a handoff note that preserves goal, status, risks, and the next prompt.

## 7. Platform-Specific “Fixed” Claims

Use AskProof when Cursor, Claude Code, Codex, or a domestic AI agent platform says work is fixed.

AskProof should still ask:

- What was changed?
- What was verified?
- What was skipped?
- Does the evidence prove the user-facing problem?

See [Example Gallery](example-gallery.md).

## 8. Role-Specific Prompt Rescue

Use AskProof when product managers, designers, operations teams, or content creators give vague but natural feedback.

Examples:

- Product manager: “这个流程还是不对，继续优化。”
- Designer: “这个页面不好看，重做。”
- Operations: “这个自动化还是不稳定，你自己继续修。”
- Content creator: “生成结果还是不对，继续改。”

AskProof should preserve the user’s intent while turning the feedback into a more actionable prompt.
